## Genuine Fable Failures

### 002-queries/022-unbounded_query_no_collect

**Classification**: MODEL_FAULT

This failed in all three runs. Fable avoided `.collect()` by using `.take(100)`, but missed the required newest-first ordering. The grader expected descending `_creationTime` order.

### 004-actions/005-storage_http_action

**Classification**: MODEL_FAULT

One run used `req.bytes()` in the upload handler. The reference solution uses `request.blob()`, and the POST storage tests failed.

### 003-mutations/004-cascade_delete

**Classification**: MODEL_FAULT

One run returned `null` when deleting a missing user. The grader expects an error matching `/not found/i`.

## Actions Taken

1. Added `compareFunctionSpec` checks to graders that were trying to assert function kind or visibility by making invalid runtime calls.
2. Removed wrong-kind query/mutation/action calls that can destabilize the Convex client and turn a clean failure into a timeout.
3. Removed regular-client calls to internal function refs where privacy is already covered by function spec comparison and admin-client checks.
4. Changed `003-mutations/007-batch_delete_self_schedule` to inspect the `activityLog` table through the admin `_system/frontend/listTableScan` API instead of probing optional public helper queries.
