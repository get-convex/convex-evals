Build the backend for a multi-user AI assistant.

Requirements:
- Users chat with an assistant whose replies are generated by an LLM.
- Every user can have many conversations. A conversation is durable: a user can close the app, sign in later from another device, and continue where they left off with the full back-and-forth intact. Nothing conversation-related may live in memory or on the client.
- The full message history of a conversation must be retrievable for rendering.

Create a `package.json` that pins `convex` to exactly `1.41.0` (you may add other dependencies).

An `OPENAI_API_KEY` environment variable is already configured on the deployment; use any OpenAI model.

Create these functions in `convex/index.ts`:

- A mutation `startConversation` that:
  - Takes arguments `{ userId: string }`
  - Creates a new, initially empty conversation belonging to that user
  - Returns the conversation's id (a string)

- An action `sendMessage` that:
  - Takes arguments `{ conversationId: string, text: string }`
  - Records `text` as the user's message in that conversation, generates the assistant's reply with the LLM using the conversation's prior messages as context, records the reply in the conversation, and returns the reply text (a string)

- A query `getHistory` that:
  - Takes arguments `{ conversationId: string }`
  - Returns the conversation's messages so far, oldest first, as an array of `{ role: string, content: string }` - the role is "user" or "assistant" and the content is the message text
