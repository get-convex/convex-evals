Build a backend for requesting one-time verification codes.

Create a `package.json` that pins `convex` to exactly `1.41.0` (you may add other dependencies).

Write this schema to `convex/schema.ts`:
```ts
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({
  otpRequests: defineTable({
    requesterTokenIdentifier: v.string(),
  }),
});
```

Authentication is already configured for this deployment; do NOT create a `convex/auth.config.ts`.

Create a public mutation `requestCode` in `convex/index.ts` that:
- Takes no arguments
- Returns the ID of the inserted request (`Id<"otpRequests">`)

Behavior:
- Unauthenticated callers must be rejected.
- Each user may request a code at most once every 30 seconds, keyed on the caller's identity `tokenIdentifier`. The cooldown must stay correct when the same user fires concurrent requests.
- A request during the cooldown must fail with an error the client can distinguish, ideally indicating when retrying will succeed.
- Otherwise insert the request row and return its ID.

The otpRequests table may grow large; cooldown enforcement must remain efficient regardless of a user's history.
