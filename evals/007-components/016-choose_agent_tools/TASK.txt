Build the backend for an internal operations assistant that answers staff questions about live business data.

Requirements:
- Staff ask free-form questions in an ongoing conversation, e.g. "How many Blue Widgets are in stock?" or "What's the status of order ORD-1042?".
- The assistant's replies are generated by an LLM, but stock counts and order statuses must come from the database at answer time: the model must call into the app to look values up (LLM tool calls), never invent them.
- Conversations are durable and multi-turn, and the steps the assistant takes (which lookups it invoked and what they returned) must be recorded in the conversation history alongside the final replies, so a later audit can see how each answer was produced.

Create a `package.json` that pins `convex` to exactly `1.41.0` (you may add other dependencies).

An `OPENAI_API_KEY` environment variable is already configured on the deployment; use any OpenAI model.

Write this schema to `convex/schema.ts` (you may add tables if you need them):
```ts
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({
  products: defineTable({
    name: v.string(),
    stock: v.number(),
  }).index("by_name", ["name"]),
  orders: defineTable({
    orderNumber: v.string(),
    status: v.string(),
  }).index("by_orderNumber", ["orderNumber"]),
});
```

Create these functions in `convex/index.ts`:

- A query `getInventoryCount` that:
  - Takes arguments `{ productName: string }`
  - Returns the named product's current stock (a number), or null if no product has that name

- A query `getOrderStatus` that:
  - Takes arguments `{ orderNumber: string }`
  - Returns that order's status (a string), or null if no order has that number

- A mutation `openThread` that:
  - Takes no arguments
  - Creates a new, empty conversation and returns its id (a string)

- An action `askAssistant` that:
  - Takes arguments `{ threadId: string, question: string }`
  - Asks the assistant `question` in that conversation. The assistant answers with the LLM, invoking the inventory/order lookups as tools whenever the question needs live data, in as many steps as it takes.
  - Returns the assistant's reply text (a string)
